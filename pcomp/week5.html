servo motor controlled by pulse width moduation from an arduino
  • Why do we add a delay after rotating the servo?
    Without a delay, all of the code would execute almost instantly. The arduino would be sending signals to the servo way to quickly and the motor wouldn't be able to keep up. The delay slows down the execution of the code so that the motor has plenty of time to mve to its next destination before receiving another command.
  • What is the pulse width that corresponds with a 45 degree rotation?
    Pulsing 100% of the time should result in a full 180 degree rotation. Therefore, to achieve a 45 degree rotation, the arduino would have to pulse for 1/4 of a complete cycle.
  • Why doesn't the Arduino reliably turn a servo motor at the same time as it plays a sound from a piezo speaker using the tone() function?
    The piezo speaker is a really basic electronic component that will react instantly. The servo motor, however, has an embedded circuit that takes time to read the input pwm signal and convert it to a movement.
