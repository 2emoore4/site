visualizing a single series of data

for the single series assignment, we decided to use mta turnstile data. this data provides weekly totals of metrocard swipes for each individual station. it splits the swipes into different categories (full fare, discounted rate, weekly pass, monthly pass, etc...), but we simply used the full fare data because it was the most populated category. this data is represented by a simple bar graph. each subway station has its own bar that shows the number of full fare swipes. since there are so many stations, we didn't want to clog the screen with text, so when you hover over one of the bars, it shows the name of the station. there is a scrollbar at the top, because it would have been tough to fit all 500 stations on the screen at the same time.

animated visualization

I found a government site that provides live data for all recent earthquakes. the feed updates once a minute (so it's not exactly live, but pretty close). obviously there aren't a ton of earthquakes at any given time so this update frequency is sufficient and it's pretty close to being live. an earthquake gets added to the feed as soon as it occurs and it disappears after it has fully subsided. as you can see from the video, each earthquake shows up as a different size. the feed also gives the magnitude of each earthquake. the bigger the circle, the greater the earthquake. I recorded this video last night, and most of the active earthquakes were in california. this is usually the case, but I actually saw a pretty big earthquake in china earlier yesterday. the blue bar at the bottom of the window fills every minute, while it's waiting to get an update. as soon as the bar fills, the data refreshes and new earthquakes are shown (if there are any). since earthquakes are fairly infrequent, I recorded the video over a span of 24 minutes, then sped it up so that you could actually see some activity without waiting a half an hour.
